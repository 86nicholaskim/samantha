# AI Server (AI서버)

## 1. 파이썬 서버 (Python Server)
- **역할**: LLM 응답값 반환 및 데이터 처리 서버.
- **기능**: 더미 데이터 처리, 브라우저와 LLM 간의 응답 중계.
- **기술 스택**: Python, FastAPI/Flask (현재 `server.py` 존재).

---

## 2. LLM 모델 선정: Gemma vs Llama 3

프로젝트의 핵심 지능인 로컬 LLM 선정을 위한 비교 분석입니다.

### 📋 모델 비교 요약

| 항목 | Google Gemma (Gemma 2) | Meta Llama 3 (3.1/3.2) |
| :--- | :--- | :--- |
| **주요 강점** | 효율적인 아키텍처, 한국어 성능 우수 | 업계 표준, 강력한 추론 및 코딩 능력 |
| **멀티모달 (Vision)** | **PaliGemma** 등 강력한 시각 분석 지원 | **Llama 3.2 (11B)** 부터 공식 지원 |
| **로컬 리소스 효율** | 2B, 9B 모델로 저사양 기기 최적화 | 8B 모델이 기본이며 성능 대비 가벼움 |
| **라이선스** | 비교적 유연함 (상업적 이용 가능) | 매우 개방적 (단, 대규모 이용 시 제한) |
| **Fine-tuning** | LoRA/Unsloth 최적화 잘 됨 | 커뮤니티 지원 및 데이터셋 풍부 |

### 🔍 Project Samantha 관점에서의 분석

1. **시각 지능 (Visual Intelligence)**:
   - 프로젝트 로드맵(Phase 2)에서 이미지 및 엑셀 분석이 필수적입니다.
   - **Gemma**: 이미 검증된 PaliGemma와 같은 시각 특화 모델 계열이 있어 유리합니다.
   - **Llama 3**: 최신 3.2 버전에서 Vision을 지원하지만, 11B 모델은 9B Gemma보다 사양이 높을 수 있습니다.

2. **인격화 (Fine-tuning)**:
   - 영화 <Her>의 사만다 대본을 학습시켜야 합니다.
   - 두 모델 모두 LoRA를 통한 파인튜닝이 원활하며, Ollama를 통해 커스텀 모델 배포가 가능합니다.

3. **로컬 성능**:
   - Ollama 환경에서 구동 시, 8B/9B급 모델이 가장 안정적입니다.
   - Llama 3 8B는 일반적인 대화 및 추론에서 최고 수준의 성능을 보여줍니다.

### 💡 최종 고민 및 제안

- **Gemma 선택 시**: "사만다"라는 브랜드와 Google 생태계의 멀티모달 확장성을 중시할 때 적합합니다. (현재 문서상 기본 계획)
- **Llama 3 선택 시**: 범용적인 지능과 한국어 대화의 자연스러움, 그리고 강력한 커뮤니티 자산을 활용하고 싶을 때 적합합니다.

> **현재 상태**: `doc/plan.md`에 따라 **Gemma**를 기본으로 설정하고 있으나, 멀티모달 구현 시점에 **Llama 3.2 Vision**과의 성능 비교 후 최종 결정 권장.
