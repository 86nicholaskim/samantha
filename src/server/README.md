# AI Server (AI서버)

## 1. 파이썬 서버 (Python Server)
- **역할**: LLM 응답값 반환 및 데이터 처리 서버.
- **기능**: 사만다의 인격 기반 대화 생성 및 자아 성장(Memory Distillation) 처리.
- **기술 스택**: Python, FastAPI.

### 🧩 주요 모듈 및 서비스
- **`core/brain.py`**: 사만다의 사고 엔진. 자아(Soul) 데이터를 로드하여 페르소나를 유지하고 대화 로직을 총괄.
- **`services/memory_manager.py`**: 대화 종료 후 인사이트를 추출하여 자아를 성장시키고, 데이터 안전을 위해 스냅샷을 생성.
- **`models/ollama_client.py`**: 로컬 Ollama API와의 통신을 담당하는 경량 클라이언트.

### 🔌 API 엔드포인트
- `POST /chat`: 사용자의 입력에 대한 사만다의 답변 생성.
- `POST /evolve`: 대화 기록을 분석하여 사만다의 기억과 성격을 업데이트 (스냅샷 생성 포함).
- `GET /`: 서버 상태 및 현재 로드된 모델 정보 확인.

## 2. LLM 모델 선정: Gemma 4 vs Llama 5 (2026년 기준)

프로젝트의 핵심 지능인 로컬 LLM 선정을 위한 최신 비교 분석입니다.

### 📋 모델 비교 요약

| 항목 | Google Gemma 4 | Meta Llama 5 |
| :--- | :--- | :--- |
| **주요 강점** | 네이티브 멀티모달, 극강의 효율성, Apache 2.0 | 초거대 컨텍스트 (5M), 자기 개선 능력 |
| **멀티모달 (Vision/Audio)** | **기본 내장** (텍스트, 이미지, 오디오, 비디오) | 초고성능 추론 기반 멀티모달 지원 |
| **로컬 리소스 효율** | E2B, E4B, 26B MoE 등 저사양 최적화 | Llama 4 Scout(17B MoE) 등 효율적 모델 존재 |
| **라이선스** | **Apache 2.0** (완전 오픈 소스) | Meta Llama License (상업적 이용 가능) |
| **핵심 기술** | Gemini 3 아키텍처 기반 | Recursive Self-Improvement (자기 강화 학습) |

### 🔍 Project Samantha 관점에서의 분석

1. **시각 및 멀티모달 지능 (Multimodal Intelligence)**:
   - **Gemma 4**: 설계 단계부터 이미지, 오디오, 비디오를 동시에 처리하는 네이티브 멀티모달로, 사만다의 "눈과 귀" 역할을 수행하기에 최적입니다.
   - **Llama 5**: 500만 토큰의 거대한 컨텍스트 윈도우를 통해 장기 대화 기억 및 방대한 데이터 분석에 압도적 우위를 가집니다.

2. **인격화 (Persona & Fine-tuning)**:
   - 영화 <Her>의 사만다 대본 학습 시, 두 모델 모두 뛰어난 성능을 보입니다. 특히 Gemma 4의 Apache 2.0 라이선스는 인격화된 모델의 배포 및 활용에 있어 제약이 없습니다.

3. **로컬 실행 환경**:
   - **Gemma 4 E4B**: 일반 소비자용 기기에서도 고속 추론이 가능하여, 프로젝트의 접근성을 높입니다.
   - **Llama 5/4**: 고성능 추론을 위해서는 Llama 4 Scout(17B MoE) 계열이 로컬 GPU 환경에서 현실적인 대안입니다.

### 💡 최종 고민 및 제안

- **Gemma 4 선택 시**: 완전한 오픈 라이선스와 네이티브 멀티모달 확장성을 중시할 때 최선의 선택입니다. (현재 기본 계획)
- **Llama 5/4 선택 시**: 방대한 컨텍스트 활용과 복잡한 논리 추론 능력이 최우선일 때 적합합니다.

> **현재 상태**: `doc/plan.md`에 따라 **Gemma 4 (gemma4:e4b)**를 기본으로 설정합니다. 네이티브 멀티모달 기능이 사만다 프로젝트의 지향점과 일치합니다.
